
# Casa Aurora — Projeto de Plataforma (Entrega de Unidade)

Projeto simples e acessível para uma ONG de apoio a pessoas trans. O aluno deverá subir este código em um repositório público no GitHub e enviar o link conforme instruções da disciplina.

## Estrutura
- index.html — página inicial (institucional)
- projetos.html — listagem de projetos e chamadas ao voluntariado
- cadastro.html — formulário complexo para voluntariado/contato
- assets/css/style.css — estilos (mobile-first, responsivo)
- assets/js/main.js — máscaras de input (CPF, telefone, CEP) e pequenas melhorias de acessibilidade
- assets/img/* — placeholder para imagens (substituir)

## Observações para entrega
1. O repositório **deve** ser público no GitHub. Suba a pasta inteira e habilite GitHub Pages (branch `main` ou `gh-pages`) se quiser gerar um link público estático.
2. Todos os HTML usam marcação semântica e atributos para acessibilidade (skip link, landmarks, aria-*).
3. Validação HTML: os arquivos foram escritos para seguir boas práticas do HTML5; é recomendável validar no W3C Validator antes da entrega final.
4. O usuário (aluno) é responsável por inserir as mídias finais (imagens e vídeos otimizados).

## Como usar
- Teste localmente abrindo qualquer .html no navegador.
- Para simular envio no formulário, apenas clique em enviar — ele mostra um alerta (demo) e não envia dados.

## Checklist obrigatório (para correção)
- [x] Mínimo 3 páginas HTML semânticas
- [x] Formulário com campos exigidos: nome, e‑mail, CPF, telefone, data de nascimento, endereço, CEP, cidade, estado
- [x] Validações nativas HTML5 + padrões de máscara para inputs
- [x] Responsividade e elementos focusáveis (skip link)
- [x] Instruções para publicar no GitHub
